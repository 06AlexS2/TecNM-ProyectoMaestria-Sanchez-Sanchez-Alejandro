ESTE ARCHIVO CONTIENE LAS INDICACIONES NECESARIAS PARA LA REPLICACIÓN/REUTILIZACIÓN DEL PROYECTO EN TRABAJOS FUTUROS.

LA CARPETA CONTIENE DIVERSOS ARCHIVOS, TODOS NECESARIOS PARA EL FUNCIONAMIENTO DEL VIDEOJUEGO, Y SE DESCRIBEN DE LA SIGUIENTE FORMA:

1. shootermk4sinbotones.py : ESTE ES EL ARCHIVO PRINCIPAL DEL JUEGO, PARA PODER CORRERLO, SE NECESITA INSTALAR, PREVIAMENTE EN UN ENTORNO VIRTUAL,
LAS LIBRERIAS LISTADAS AL INICIO DEL SCRIPT. DE LO CONTRARIO, NO FUNCIONARÁ.

2.randomized_selector.py : SCRIPT PARA SELECCION Y ASIGNACION DE PARTICIPANTES EN LOS GRUPOS PROTOCOLARIOS

3.archivos de niveles "level_n_data.csv" : EN CADA ARCHIVO, DONDE n ES UN NUMERO DEL 1 AL 14, EXISTEN LOS DATOS DE JUEGO, NECESARIOS PARA
PODER RENDERIZAR LOS niveles

4.CARPETA ./audio : CONTIENE LOS ARCHIVOS DE AUDIO REPRODUCIDOS EN EL VIDEOJUEGO. ES IMPORTANTE RESPETAR LA RUTA TAL Y COMO SE PRESENTA,
DE LO CONTRARIO, CAMBIARLO EN EL SCRIPT #1.

5. CARPETA ./DAC : CONTIENE EL SCRIPT DE ARDUINO NECESARIO PARA RECREAR EL PROTOTIPO DE CONTROL. EL ARCHIVO EN CUESTION SE LLAMA
"GamepadMK1.ino". SUBIR ESE ARCHIVO AL ARDUINO UTILIZADO (EN CASO DE SEGUIR UTILIZANDO ARDUINO).

6. CARPETA ./img : EN ELLA, EXISTEN OTRAS CARPETAS QUE CONTIENEN LOS RECURSOS GRAFICOS RENDERIZADOS EN EL VIDEOJUEGO.
SEGUIR MISMA INSTRUCCION QUE EN EL PUNTO 4.

7. CARPETA ./loggerdata : POR DEFECTO, ESTA CARPETA DEBE ESTAR, CUANDO MENOS, CREADA ANTES DE IMPLEMENTAR EL SCRIPT DEL PUNTO 1. EN ELLA,
SE IRAN ALMACENANDO LOS REGISTROS DE SENSORES Y GAMEPLAY EN CADA SESION DE JUEGO MEDIANTE ARCHIVOS CSV.

8.CARPETA ./MapArch : NO ES NECESARIA, SIN EMBARGO, AHI SE ALMACENAN LOS ARCHIVOS .tmx PARA LA EDICION DE NIVELES EN LA APLICACION "TILED"

9. button.py : EN LA VERSION FINAL, ESTE ARCHIVO NO FUE NECESARIO, SIN EMBARGO, SI SE QUIERE CREAR BOTONES GRAFICOS DE MENU PRINCIPAL,
ESTE ARCHIVO PERMITE ESO.
